statements vs expressions

statements in rust are instructions that perform an action. they do not return a value. expressions evaluate something and return a value.

statements end with a semicolon, except for function definitions. expressions do not.

fn example() -> u32 {
    let x = 2; // this is a statement
    let y = {
        x + 2 // this is an expression
    };

    3 // a function in rust will return its last expression, which is 3 here
} // the whole function definition is a statement, too

fn main() {
    println!("{}", example());
}

if you add a semicolon at the end of an expression, it will become a statement and not return anything.
