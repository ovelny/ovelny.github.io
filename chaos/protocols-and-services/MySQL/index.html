MySQL

Main points

  • Open-source SQL Relational Database Management System (RDBMS).
  • Use port TCP 3306.
  • Supported by oracle.
  • Client-server principle: a MySQL server able to talk to one or many MySQL clients.
  • Databases are stored in .sql files.
  • Suited for dynamic websites.
  • Often combined with Linux, Apache, PHP (LAMP), or with nginx instead of Apache (LEMP).
  • MariaDB is a fork of MySQL, developed after Oracle acquired MySQL.

Default system databases

Default system database Description
mysql System database storing information for mysql server.
information_schema Provide access to database metadata.
performance_schema Monitor mysql server execution at a low level.
sys Set of objects to interpret data from performance_schema.

Insecure configuration

Configuration can often be found in /etc/mysql/mysql.conf.d/mysqld.conf. Many things can go wrong and end up misconfigured, but the main settings to look for from a security standpoint are user, password, admin_address, debug, sql_warnings and secure_file_priv.

Commands

MySQL command Description
mysql -u <user> -p<password> -h <IP address> Connect to MySQL server.
show databases; Show all databases.
use <database>; Select a database.
show tables; Show all tables in the database.
show columns from <table>; Show all columns in the database.
select * from <table>; Show everything in the table.
select * from <table> where <col> = "<string>"; Search for string in table.
quit Exit MySQL.
show variables like "secure_file_priv"; Check FILE privilege.
select "<webshell>" into outfile <path> Add a new file as webshell.
select LOAD_FILE("<file path>") Read local file. 
# Scan the MySQL server:
sudo nmap -sV -sC -p 3306 --script mysql* <IP address>
