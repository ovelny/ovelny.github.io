r-services

main points

  • suite of services for remote access or commands
  • used before the SSH protocol
  • unencrypted format, making it vulnerable to MITM attacks
  • use ports TCP 512, 513 and 514

the r-commands suite has the following programs:

command description
rcp remote copy
rexec remote execution
rlogin remote login
rsh remote shell
rstat remote stats
ruptime remote uptime
rwho remote who

insecure configuration

check the /etc/hosts.equiv and ~/.rhosts files for a list of trusted hosts who are automatically granted access without authentication. these files use the + symbol as a wildcard to allow any external user and / or any host to execute r-commands.

commands

# scan r-services ports with nmap
nmap -sV -p 512,513,514 <IP address>

# logging in with rlogin
rlogin <IP address> -l <username>

# once logged in, list all interactive sessions
rwho

# list authenticated users
rusers -al <IP address>
