Xfreerdp

Xfreerdp works reliably but can be slow at times. Still, I'd rather use this tool rather than remmina or others because I prefer a command-line interface.

Here's an optimized command that helps alleviate latency issues and enforce an azerty keyboard on the target:

xfreerdp /u:<username> /p:<password> /v:<IP> /dynamic-resolution /kbd:0x0000040C /cert-ignore /bpp:8 /network:modem /compression -themes -wallpaper /relax-order-checks /auto-reconnect -grab-keyboard +clipboard

It doesn't make everything run smoothly but it still helps a lot!

Socks proxy

When using proxychains, xfreerdp is usable by prepending the command like you normally would:

proxychains xfreerdp /u:<username> /p:...

But if you use x11 forwarding, proxychains will fail to interpret the localhost display variable. What works is to use xfreerdp's proxy argument instead:

xfreerdp /u:<username> /p:<password> /proxy:socks5://127.0.0.1:1080 /v:<target IP address> ...

Windows Server 2008 and old systems

On old systems such as Windows Server 2008, you might need to add the /sec:rdp flag before all the other flags.
