Immutability vs mutability vs shadowing

Variables in Rust are immutable by default:

fn main() {
    let x = 0;
    x = 1; // will not compile
}

To modify an existing variable, you can either declare it as mutable or shadow it.

Mutable variables

To make a variable mutable, just use the mut keyword after let:

fn main() {
    let mut x = 0;
    x = 1; // will compile
}

Shadowing a variable

We can also re-declare a variable, so that it shadows the previous one:

fn main() {
    let x = 0;
    let x = 1;
    println!("{}", x); // will compile and output 1
}

So why do we have different ways of changing a variable?

  • Sometimes, we might want to change a variable and keep it immutable afterwards (shadowing).
  • We might want to change a variable but make sure it keeps the same type as a safety check (mutable).
  • On the contrary, we might want to change a variable's type along the way and keep its name (shadowing).
